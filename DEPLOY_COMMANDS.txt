===============================================================================
                     VOLO PLATFORM - DEPLOYMENT COMMANDS
===============================================================================

STEP 1: CREATE GITHUB ACCOUNT & REPOSITORY
-------------------------------------------
1. Go to https://github.com and create a free account
2. Click "New repository"
3. Name: "volo-platform"
4. Description: "Volo Hotel Booking Platform with LiteAPI Integration"
5. Make it Public or Private (your choice)
6. Click "Create repository"

STEP 2: PREPARE PROJECT FOR DEPLOYMENT
----------------------------------------
# Navigate to project directory
cd volo-platform

# Install dependencies
npm install

# Create production build
npm run build

# Check for any errors
npm run lint

# Create .env.local with your credentials
cp .env.example .env.local
# Edit .env.local and add your real values

STEP 3: PUSH CODE TO GITHUB
-----------------------------
# Initialize git repository
git init

# Add all files
git add .

# Create initial commit
git commit -m "Initial commit: Volo Platform with LiteAPI integration"

# Set main branch
git branch -M main

# Add remote repository (replace YOUR_USERNAME)
git remote add origin https://github.com/YOUR_USERNAME/volo-platform.git

# Push to GitHub
git push -u origin main

STEP 4: DEPLOY TO VERCEL
-------------------------
1. Go to https://vercel.com
2. Sign up with your GitHub account
3. Click "New Project"
4. Find and select "volo-platform" repository
5. Click "Import"

# Vercel Project Settings:
Framework Preset: Next.js
Root Directory: ./
Build Command: npm run build
Output Directory: .next
Install Command: npm install

6. Click "Deploy"

===============================================================================
                              TROUBLESHOOTING
===============================================================================

COMMON ISSUES:

1. Build Errors:
   - Check npm run build locally first
   - Ensure all environment variables are set
   - Check TypeScript errors

2. API Not Working:
   - Verify LiteAPI keys in environment variables
   - Check API logs in Vercel dashboard
   - Test /api/health endpoint first

3. Database Connection Issues:
   - Update DATABASE_URL with correct connection string
   - Ensure database allows connections from Vercel

4. Environment Variables:
   - Double-check all variables are set in Vercel dashboard
   - No quotes in environment variable values
   - Use correct variable names (case-sensitive)

5. LiteAPI Integration:
   - Verify private and public keys are correct
   - Check API endpoint URL is accessible
   - Monitor API usage limits
